## What is "Attacks by Service"?

**"Attacks by Service"** is a horizontal bar chart in the Dashboard that shows **which honeypot services are being attacked** and **how many attacks each service received**.

---

## ğŸ¯ What It Shows

The chart displays:
- **Service Name** (on the left)
- **Number of Attacks** (bar length)
- **Visual comparison** of attack distribution across services

---

## ğŸ¯ The Three Honeypot Services

Your system has **3 honeypot services** that attackers interact with:

### 1. **Fake Git Repository** (Port 8001)
- **File:** `scripts/fake_git_repo.py`
- **What it simulates:** A Git repository server
- **Common attacks:**
  - `git_push` - Malicious commits
  - `file_access` - Accessing sensitive files (.env, secrets.yml)
  - `bruteforce_login` - Login attempts
  - `malformed_payload` - SQL injection, XSS attempts

### 2. **Fake CI/CD Runner** (Port 8002)
- **File:** `scripts/fake_cicd_runner.py`
- **What it simulates:** A CI/CD pipeline system (like Jenkins)
- **Common attacks:**
  - `ci_job_run` - Executing malicious jobs
  - `ci_credentials_access` - Stealing credentials
  - `bruteforce_login` - Login attempts

### 3. **Consolidated Honeypot Services** (Port 8000)
- **File:** `Honeypot/honeypot_services.py`
- **What it simulates:** Combined Git + CI/CD services
- **Common attacks:**
  - All Git repository attacks
  - All CI/CD attacks
  - `scan_attempt` - Port scanning

---

## ğŸ“Š How the Chart Works

### Data Source
The chart gets data from the **logging server** (`/stats` endpoint):

```sql
SELECT target_service, COUNT(*) as count 
FROM logs 
GROUP BY target_service 
ORDER BY count DESC 
LIMIT 10
```

### Chart Display
- **Horizontal bars** showing attack counts
- **Color-coded** bars (red, orange, yellow, green, blue)
- **Sorted by count** (highest first)
- **Updates every 5 seconds** (dynamic)
